# my_health_app - 服薬・健康管理＆AIアシストアプリ

「日常の服薬管理」と「AIによる健康相談」、さらに「周辺の医療機関検索」を1つに統合した、日々の健康をサポートするFlutter製アプリケーションです。

## 開発の背景と目的
「自分や家族の薬の飲み忘れを防ぎたい」「日々のちょっとした健康や食事の悩みを、手軽に専門家視点で相談できるツールが欲しい」という身近な課題を解決するために本アプリを開発しました。

単なるリマインダーにとどまらず、生成AIを活用したアドバイス機能や、いざという時に役立つ周辺マップ機能を搭載することで、総合的な健康サポートアプリを目指しました。

---

## 主な機能

### 1. 服薬カレンダー＆記録機能 (Firestore連携)
* カレンダーUI: `table_calendar` を用いて日々の服薬状況を視覚的に管理。
* クラウド同期: チェックリストのチェック状態はリアルタイムで Firebase Cloud Firestore に保存・同期され、端末を変えてもデータが引き継がれます。

### 2. 確実な服薬リマインダー機能 (ローカル通知)
* 柔軟なスケジュール: 朝・昼・晩など、ユーザーのライフスタイルに合わせた通知時間を設定可能。
* バックグラウンド通知: `flutter_local_notifications` を使用し、アプリを閉じていても指定した時間に確実にプッシュ通知を届けます。

### 3. AI健康アドバイザー機能 (Gemini API)
* リアルタイムチャット: Googleの生成AI「Gemini 2.5 Flash Lite」モデルを搭載。
* 専門的アドバイス: ユーザーが食事内容や体調を入力すると、AIが「栄養士」のペルソナとして、短く的確なアドバイスを即座に返信します。

### 4. 周辺医療機関マップ機能 (OpenStreetMap)
* 現在地取得とマップ表示: `geolocator` で現在地を取得し、`flutter_map` で現在地周辺の地図を描画します。
* 検索・お気に入り機能: 近くの病院や薬局を検索し、ピンをタップしてウェブサイトを開いたり、お気に入りとして登録（内部ストレージに保存）することが可能です。

## 技術的な工夫と課題解決（アピールポイント）

本アプリの開発において、特にこだわった点や、直面した課題の解決プロセスです。

### 1. 最新Android OSにおける厳格な通知権限への対応
確実な服薬リマインダーを実現するためには、指定した時刻に正確に通知を発火させる必要があります。しかし、Android 12/13以降では通知権限（`POST_NOTIFICATIONS`）や正確なアラーム権限（`SCHEDULE_EXACT_ALARM`）の取得が厳格化されていました。
これに対し、`AndroidManifest.xml` の適切な記述に加え、アプリ起動時に権限リクエストのダイアログを段階的に表示するロジックを実装し、OSの仕様変更に適合した堅牢な通知システムを構築しました。

### 2. OpenStreetMap規約準拠による403エラーの解決
マップ描画時にタイルサーバーから「403 Forbidden」エラーが返される問題に直面しました。原因を調査した結果、OpenStreetMapの利用ポリシーにおいて適切な `User-Agent` の設定が求められていることが判明しました。
そこで、`flutter_map` の設定にアプリ固有のパッケージ名（例：`com.example.my_health_app`）を `User-Agent` として明記する対応を行い、サードパーティAPIの利用規約を遵守した形でのエラー解決を実現しました。

### 3. APIキー等、機密情報のセキュアな管理
Gemini API を利用するためのシークレットキーをハードコードして GitHub に公開してしまうリスクを防ぐため、キー情報を `secrets.dart` という別ファイルに分離しました。さらに、そのファイルを `.gitignore` に登録することで、**バージョン管理システムから機密情報を確実に除外するセキュリティ対策**を講じています。

### 4. 非同期処理と状態管理の適切な使い分け
Firestore からのデータ取得には `StreamBuilder` を用いてリアルタイムなUI更新を実現し、API通信や現在地取得などのワンタイムの処理には `FutureBuilder` や `async/await` を活用するなど、要件に合わせた適切な非同期処理の実装を心がけました。
